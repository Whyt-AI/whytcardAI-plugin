# /research

Perform dual-angle research on a topic before making decisions.

## Instructions

Follow the WhytCard research-first protocol:

1. **Define the question precisely** — What exactly do we need to know?
2. **Good angle search** — Official docs, best practices, who does it best (named examples)
3. **Bad angle search** — Complaints, pitfalls, anti-patterns (Reddit, GitHub issues)
4. **Compare with data** — Stars, downloads, benchmarks, bundle size
5. **Verify versions live** — Never trust training data
6. **Format output** as: What works → What fails → Recommendation → What we reject → Sources

Pass the user's topic as the research subject. Be thorough. Name real products and cite real sources.
